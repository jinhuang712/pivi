# 约束与非目标

## 关键约束

- 不部署业务云后端（用户账号、房间状态、消息记录均不落云）
- 房主设备承担频道控制与可选媒体中转
- 初期不保证所有公网 NAT 环境下都能一次连通
- 优先桌面端，不覆盖移动端

## 工程约束

- 以跨平台优先：macOS + Windows 同步迭代
- 以成熟实时协议优先：WebRTC + Opus
- 避免高维护成本：减少自研实时传输协议
- 默认离线可启动：无外部依赖可在局域网内工作

## 非目标（MVP 不做）

- 摄像头视频通话
- 大规模公会系统、跨房间社交网络
- 全功能 IM 系统（历史消息、除图片外的任意文件传输、已读回执）
- 多房间并发主持与全局调度

## 已知妥协

- 纯无云模式下，跨网连接成功率低于云中继方案
- 纯无云发现机制在当前阶段优先保证局域网可用性
- 房主主动关闭房间时，系统弹窗二次确认，确认后全体成员掉线
- 房主意外掉线将导致房间不可用（需主动执行“移交房主”才能平滑过渡）
- 公网场景需要用户具备基础网络操作能力（端口映射等）
