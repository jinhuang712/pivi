# 桌面客户端设计（macOS / Windows）

## 客户端职责

- 呈现频道与成员 UI
- 管理本地音频设备
- 执行信令握手与媒体连接
- 提供房间内文本与图片消息收发
- 提供屏幕共享采集与推送控制
- 提供全局快捷键管理与状态提示
- 展示网络质量面板（主页一键开关）与错误反馈

## 建议技术栈

- **桌面壳与后端逻辑**：Tauri + Rust（取代 Electron，极大降低电竞场景下的内存与 CPU 占用，并支持打包为单文件 `.exe` 和 `.dmg` 以便社群一键转发）
- 前端：React + TypeScript
- 状态管理：轻量 Store（Zustand/Redux 二选一）
- 实时能力：WebRTC API

## 模块划分

- `ui-shell`：窗口、路由、主题、全局快捷键绑定（全局按键需底层 API 支持）
- `user-identity`：本地 UUID 生成、昵称持久化存储与修改
- `audio-manager`：设备枚举、输入输出切换、增益控制
- `rtc-session`：PeerConnection 生命周期管理
- `chat-channel`：房间内消息（文本/图片）发送、接收与本地临时缓存
- `screen-share`：桌面源选择、系统音频混合、推流状态管理
- `channel-state`：成员状态、发言状态、权限状态
- `network-health`：延迟、丢包、重连状态、主页网络面板视图

## 交互主流程

1. 用户输入房间信息并发起加入
2. 客户端完成鉴权与信令协商
3. 建立媒体通道并进入频道视图
4. 用户可执行静音/闭麦/PTT/设备切换

## 关键 UI 页面

- 启动页：首次启动强制设置昵称，日常启动可创建频道 / 加入频道
- 频道主页：
  - 核心区：成员列表、发言状态
    - 成员项功能：本地独立调节某人音量、本地屏蔽（屏蔽其扬声器）
  - 消息区：房间消息面板
    - 聊天框：支持通过系统剪贴板粘贴截图发送，预留图片上传按钮
    - 错误提示：超过 5MB 图片时的弹窗/气泡错误提示
  - 悬浮入口：一键打开/关闭网络质量指标面板
- 设置中心：
  - 基础设置：昵称修改
  - 音频设置：输入输出设备、麦克风/扬声器全局音量、阈值调节
  - 快捷键设置：管理全局 PTT（按键说话）、静音等快捷键
- 房主管理面板（仅房主可见）：
  - 房间设置：修改房间口令
  - 权限交接：一键将房主权限移交给房间内的其他成员
  - 成员管理：踢人、封禁操作、全局强制闭麦某人
  - 黑名单列表：查看已被封禁的成员（显示昵称/UUID），支持一键解除封禁
- 共享控制浮层：开始/停止共享、帧率与清晰度档位
- 网络诊断弹窗：连通失败原因与建议
