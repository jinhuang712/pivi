# 安全模型（MVP）

## 安全目标

- 防止未授权成员进入频道
- 保护语音内容传输机密性
- 降低本地监听端口被滥用风险

## 访问控制

- 房间 ID + 口令作为第一层门槛
- 会话 token 作为第二层校验
- 角色划分：房主（Host）与普通成员（Member）
- 房主特权：
  - 拥有解散房间、踢出成员的权限
  - 拥有手动移交房主身份给其他成员的权限
  - 拥有“全局闭麦”某个成员的权限（强制该成员静音，所有人听不到他）
  - 支持将恶意用户加入本地黑名单（基于设备或 UUID）
  - 提供专门的“房间管理面板”以查看、解除封禁状态，避免误踢
- 成员约束：仅能控制自身的麦克风与本地听到的音量（包括本地屏蔽某人的声音、单独调节某人音量）

## 传输安全

- **WebRTC 默认加密**：所有音视频媒体流均通过 DTLS-SRTP 强制加密。
- **信令防篡改**：基于 WebSocket/TCP 的信令交互需做基础的数据校验，防止伪造其他用户的事件。
- **资源限制**：为防止房主节点被恶意请求打挂，图片传输建议走 WebRTC DataChannel。设置单张图片大小上限为 5MB（若用户发送/粘贴的截图超限，客户端需在本地直接拦截并给予明确的“图片超限”错误提示），信令面做严格频控。

## 本地安全策略

- 默认仅绑定必要网卡与端口
- 可配置仅局域网可见模式
- 端口冲突或异常时快速失败并明确提示

## 威胁与缓解

- 暴力尝试入房：口令错误限频与临时封禁
- 伪造成员身份：token 时效与签名校验
- 重放攻击：会话 nonce 与过期校验

## 合规与隐私

- 默认不落地语音数据
- 仅保留最小运行日志
- 提供一键清理会话数据能力
