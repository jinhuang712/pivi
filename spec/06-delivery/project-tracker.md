## 📋 功能点 Checklist (MVP)

### Phase 1: 基础设施与环境搭建 (✅ 100%)
- [x] 1.1 **代码仓库初始化**：Git, `.gitignore`。
- [x] 1.2 **项目骨架**：Tauri v2 + React 19 + TypeScript + TailwindCSS v4。
- [x] 1.3 **前端测试环境**：配置 Vitest + React Testing Library，跑通基础组件测试。
- [x] 1.4 **后端测试环境**：配置 Rust `cargo test`，支持信令结构的序列化/反序列化测试。

### Phase 2: UI 界面与交互原型 (✅ 100%)
- [x] 2.1 **侧边栏 (Sidebar)**：房间头部 (含重新生成口令)、当前用户信息卡片、成员列表占位。
- [x] 2.2 **邀请码验证组件原型**：独立字符输入框，支持自动跳格、大写转换与 `Ctrl+V` 粘贴解析（后续升级为统一 16 位邀请码输入）。
- [x] 2.3 **全局弹窗系统**：
  - [x] 加入房间的“二次确认”弹窗 (显示房间名与在线人数)。
  - [x] 设置面板模态框 (包含 Tab 切换逻辑)。
- [x] 2.4 **设置中心面板**：
    - [x] 我的账号 Tab (修改昵称与展示 UUID)。
    - [x] 语音与设备 Tab (输入/输出设备选择，VAD/PTT 模式切换)。
    - [x] 快捷键 Tab (全局 PTT 与静音快捷键绑定)。
    - [x] 封禁与黑名单 Tab (解封操作)。
- [x] 2.5 **主交互区 (Main Area)**：
  - [x] 顶部导航与网络状态指示器 (Ping, P2P/Relay 状态)。
  - [x] 屏幕共享预览占位区。
  - [x] 底部聊天框 (Chatbox)，包含 5MB 错误提示气泡。
  - [x] 主区底部中置控制条（共享开始/停止、共享画质、麦克风开关与音量、扬声器开关与音量、设置入口）。
- [x] 2.6 **成员列表高级交互**：
  - [x] 房主与普通成员状态标识 (说话指示灯)。
  - [x] 悬浮时的本地独立音量滑块与本地屏蔽(🔇)按钮。
  - [x] 房主右键菜单 (移交房主、全局闭麦、踢人、加入黑名单)。

### Phase 3: 核心网络与房主运行环境 (Host Runtime) (✅ 100%)
- [x] 3.1 **信令基础结构**：Rust `SignalingMessage` (JoinRoom, LeaveRoom, Mute 等) 及其 Serde 解析。
- [x] 3.2 **无云入口发现首版**：实现房间入口映射与查询的原型能力（后续由统一邀请码替代）。
- [x] 3.3 **WebSocket Server**：房主本地启动 WS 服务，监听指定端口。
- [x] 3.4 **房间生命周期管理**：成员加入/离开的内存状态维护。
- [x] 3.5 **鉴权与安全拦截**：基础邀请码校验逻辑与 UUID 黑名单拦截机制。
- [x] 3.6 **信令广播机制**：`ROOM_STATE`, `MEMBER_JOINED`, `MEMBER_LEFT`, 状态同步广播。

### Phase 4: WebRTC 媒体路由与通信 (✅ 100%)
- [x] 4.1 **WebRTC 协商核心 (P2P Mesh)**：Offer / Answer / ICE Candidate 的信令透传与连接建立。
- [x] 4.2 **本地设备采集**：基于浏览器 API 获取麦克风音频流。
- [x] 4.3 **音频流播放与控制**：
  - [x] 将远端音频流绑定到 `<audio>` 标签。
  - [x] 实现本地屏蔽 (Local Mute，音量置 0)。
  - [x] 实现房主全局闭麦 (Server Mute，拒绝转发及信令通知)。
- [x] 4.4 **网络降级 (Relay Fallback)**：当 P2P ICE 穿透失败时，自动将媒体流切向房主中转节点。

### Phase 5: 高级功能与产品特性 (✅ 100%)
- [x] 5.1 **数据通道 (DataChannel)**：建立高可靠的 RTCDataChannel。
- [x] 5.2 **图文消息发送**：文本广播与图片（<5MB）的二进制 P2P 分发渲染。
- [x] 5.3 **屏幕共享**：桌面/窗口流的采集与 WebRTC 推流。
- [x] 5.4 **房主移交 (Host Migration)**：基于信令的状态机流转，全员断线重连至新房主节点。
- [x] 5.5 **快捷键支持 (Global Hotkeys)**：Tauri 全局 PTT (按键说话) 与静音快捷键绑定。

### Phase 6: 打包与分发 (✅ 100%)
- [x] 6.1 **macOS 构建**：打包输出 `.dmg` 文件。
- [x] 6.2 **Windows 构建**：打包输出独立 `.exe` 文件。
- [x] 6.3 **资源优化**：图标配置，清理无用依赖，确保极低内存占用。

### Phase 7: 统一邀请码与连接稳定性（✅ 100%）
- [x] 7.1 **统一邀请码协议落地**：实现 16 位自描述邀请码编码、解码、TTL 与校验能力。
- [x] 7.2 **Join UI 单入口**：加入页只保留统一邀请码输入与分组粘贴体验。
- [x] 7.3 **建房就绪门禁**：Host Runtime 在监听、自检、端口映射完成前不展示邀请码。
- [x] 7.4 **入口稳定性增强**：完成固定端口池、历史成功端口复用与 `UPnP / NAT-PMP / PCP` 探测/映射首版接入；Windows 发布版对 NAT 映射采用兼容降级。
- [x] 7.5 **入房提速与失败回退**：完成入口预检、短超时快速重试、错误分层提示与 Relay Fallback 首版联动。
