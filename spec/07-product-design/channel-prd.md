# 频道主页 (Channel UI) 详细产品需求

## 1. 页面定位

频道主页是用户进入房间后长时间停留的核心工作区，主要承载**语音通话状态感知**、**轻量图文沟通**以及**屏幕共享观看**三大职责。

## 2. 核心模块与交互说明

### 2.1 左侧边栏：成员与身份区

- **房间头部**：显示当前房间名称（如“周末电竞开黑房”），支持点击复制 6 位字母+数字的房间口令（Room Code）。
  - **重新生成口令**：房主悬浮时可见“重新生成”按钮，点击后立即生成新的 6 位口令。
- **当前用户信息卡片**：
  - 显示当前用户头像、昵称与在线状态。
  - 不承载全局控制按钮，避免与主交互区争夺视觉焦点。
- **在线成员列表**：
  - **展示规则**：按角色排序（房主置顶，并带有特殊 Icon 标识），其余按加入时间排序。
  - **状态指示灯**：用户发言时，头像旁边的绿色指示灯呈呼吸/闪烁状态（基于 WebRTC Audio Level 映射）。
  - **成员悬浮菜单 (Hover State)**：
    - 当鼠标悬浮在某个其他成员上方时，显示**本地屏蔽按钮 (🔇)**和**本地音量滑块**。
    - **本地独立音量调节**：滑块范围 0% - 200%，调整结果仅在当前客户端本地生效，不影响其他人的听感（实现原理为调整本地 WebRTC 对应远端流的 GainNode）。
  - **房主右键菜单**（仅房主可见）：
    - “移交房主”：选择此项触发房主迁移流程，通知目标成员的客户端在本地拉起 Host Runtime 进程，准备就绪后全员断线重连。
    - “全局强制闭麦”：点击后，房主 Runtime 停止转发该用户音频，所有人都听不到他。
    - “踢出房间”：断开该用户的连接。
    - “加入黑名单”：断开该用户的连接并将其 UUID 写入本地黑名单。

### 2.2 右侧主区：屏幕共享与消息区

- **顶部导航条**：
  - 包含当前视图切换（如“屏幕共享 📺”标签）。
  - **网络状态看板开关**：位于右上角，默认折叠为简易胶囊（如 `Ping: 24ms | P2P直连`）。点击后可展开详细面板，展示当前连接模式（P2P/中转）、丢包率、延迟抖动等底层指标。
- **中置底部控制条（Google Meet 风格）**：
  - 位置：消息输入框上方、主区底部居中。
  - 控制项：屏幕共享开始/停止、共享画质切换（720p / 1080p / 原画）、麦克风开关、麦克风音量滑杆、扬声器开关、扬声器音量滑杆、设置入口。
  - 交互：共享开启时默认进入 PIP 角落预览，点击预览后可展开，并支持“缩小至角落”回退。
- **屏幕共享占位区 (Screen Share View)**：
  - 无人共享时，显示占位 UI。
  - 共享开启时，画面先以 PIP 形式呈现，再由用户主动展开为主视图。
- **底部消息面板 (Chatbox)**：
  - **消息列表**：展示文本与图片气泡，附带发送者昵称与时间戳。
  - **输入框交互**：
    - **文本输入**：支持回车发送，Shift+回车换行。
    - **图片发送 (核心机制)**：
      - 支持点击左侧 `⊕` 图标通过系统文件选择器上传图片。
      - 强烈支持通过 `Ctrl+V / Cmd+V` 将剪贴板中的截图直接粘贴至输入框。
      - **异常拦截**：前端在读取图片文件时，若文件体积 > 5MB，立刻阻止发送动作，并在输入框上方弹出红色的“⚠️ 图片大小不能超过 5MB”气泡提示（3秒后自动消失）。
