# 设置与房主管理中心 (Settings & Host Panel) 详细产品需求

## 1. 页面定位

该页面用于处理偏好设置、硬件设备调试以及房主专属的房间管理操作。在桌面端，它通常以全屏覆盖弹窗（Modal）的形式出现。

## 2. 核心模块与交互说明

### 2.1 侧边栏菜单

菜单分为两组：“用户设置”与“房主管理面板”（后者仅当当前用户为该房间房主时可见）。

### 2.2 用户设置区 (所有成员可见)

- **我的账号**：
  - 支持修改本地昵称（即时生效，持久化保存于本地，并广播给房间内其他人更新 UI）。
  - 显示系统自动分配的本地唯一 UUID（仅用于故障排查和黑名单识别）。
- **语音与设备**：
  - **输入设备 (麦克风)**：下拉列表枚举系统可用录音设备，支持热插拔自动刷新。
  - **输出设备 (扬声器)**：下拉列表枚举系统可用播放设备。
  - **输入模式选择**：
    - **语音激活 (VAD)**：默认模式，根据声音分贝自动开关麦克风。支持调节“激活阈值”滑块，滑块下方带有一个实时的声音强度指示条用于测试。
    - **按键说话 (PTT)**：选择此项后，麦克风平时保持静音，仅在按下指定快捷键时激活。
- **快捷键设置**：
  - 提供绑定系统级全局快捷键的 UI。
  - 核心支持事件：全局 PTT 按键绑定、全局静音开关。
  - **注意**：此处需判断系统权限（如 macOS 的辅助功能权限），若无权限则给出明确的引导授权提示。

### 2.3 房主管理面板 (仅房主可见)

- **房间设置**：
  - 允许修改当前房间的连接口令，修改后新加入的成员需使用新口令，已在房间内的成员不受影响。
- **封禁与黑名单 (Blacklist)**：
  - 列表展示当前被封禁的用户（显示被封禁时的昵称和对应的 UUID）。
  - **一键解封**：点击列表项右侧的“解除封禁”按钮后，将被选用户的 UUID 从内存/本地文件中移除，该用户可重新通过口令加入房间。
- **移交房主权限**：
  - 列表展示当前房间内除自己外的其他在线成员。
  - 选择一名成员并点击“移交”，将触发底层控制面的移交信令：
    1. 通知目标成员的客户端在本地拉起 Host Runtime 进程。
    2. 新进程端口准备就绪后，原房主向所有人广播新目标地址。
    3. 全体断线重连，旧房主 Runtime 销毁，权限移交完成。
