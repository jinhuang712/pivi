# 统一邀请码测试用例 (TDD)

## 1. 统一邀请码编解码

### [TC-ISC-01] 16 位邀请码编码成功
- **描述**：验证 Host Runtime 能将 `IPv4 + Port + Scope + Mode + Expiry` 编码为 16 位 Base32 邀请码。
- **测试步骤**：
  1. 输入 `203.0.113.12:7788`、`direct-host`、有效过期时间槽。
  2. 执行编码。
- **预期结果**：
  - 返回长度为 16 的邀请码原文。
  - UI 可按 `XXXX-XXXX-XXXX-XXXX` 格式展示。

### [TC-ISC-02] 16 位邀请码解码成功
- **描述**：验证加入端可以从合法邀请码恢复出原始入口。
- **测试步骤**：
  1. 使用已知样例邀请码执行解码。
  2. 读取输出字段。
- **预期结果**：
  - 正确恢复 `version / addr_family / join_mode / ipv4 / port / expiry_slot`。

### [TC-ISC-03] 校验码错误拒绝解码
- **描述**：验证任意 1 位输入错误会触发校验失败。
- **测试步骤**：
  1. 准备一个合法邀请码。
  2. 修改其中任意 1 位字符。
  3. 执行解码。
- **预期结果**：
  - 返回 `InvalidChecksum` 或等效错误。

### [TC-ISC-04] 过期邀请码拒绝加入
- **描述**：验证过期时间槽已失效的邀请码在握手前被拒绝。
- **测试步骤**：
  1. 生成一个已过期邀请码。
  2. 执行解码与连接准备。
- **预期结果**：
  - 返回 `ExpiredInviteCode`。
  - 不发起任何网络连接。

## 2. Join UI 统一输入

### [TC-ISC-05] 16 位分组输入进入统一解析流程
- **描述**：验证输入 `XXXX-XXXX-XXXX-XXXX` 时，Join UI 进入统一邀请码解码流程。
- **测试步骤**：
  1. 输入 `K7M2-9Q4P-T8XD-3F6N`。
  2. 触发加入解析。
- **预期结果**：
  - 自动去除分隔符。
  - 调用邀请码解码逻辑。

### [TC-ISC-06] 粘贴整串邀请码自动拆分
- **描述**：验证用户粘贴完整邀请码时，输入框能自动拆分并补全焦点状态。
- **测试步骤**：
  1. 向输入框粘贴 `K7M2-9Q4P-T8XD-3F6N`。
- **预期结果**：
  - 输入框按 4 组填充。
  - 光标移动到末尾。
  - 校验按钮进入可用状态。

## 3. 连接与回退

### [TC-ISC-07] 邀请码恢复地址后直连 Host Runtime
- **描述**：验证解码成功后，客户端可直接连接 `ws://ip:port`。
- **测试步骤**：
  1. 解码获得 `203.0.113.12:7788`。
  2. 发起 WebSocket 连接。
- **预期结果**：
  - 连接目标与邀请码还原值一致。
  - 成功进入后续 `JOIN_ROOM` 流程。

### [TC-ISC-08] 入口不可达提示明确错误
- **描述**：验证邀请码可解码但入口不可达时，UI 能给出明确排障提示。
- **测试步骤**：
  1. 准备一个指向不可达端口的邀请码。
  2. 发起连接。
- **预期结果**：
  - 返回“房主入口不可达”类错误。
  - UI 提示检查端口映射 / 防火墙。

### [TC-ISC-09] 直连失败时回退房主中转
- **描述**：验证邀请码连入成功后，P2P 媒体失败仍可切换到房主中转。
- **测试步骤**：
  1. 使用邀请码进入房间。
  2. 模拟 ICE 失败或超时。
- **预期结果**：
  - 触发现有 Relay Fallback。
  - 会话状态更新为 Relay。

## 4. 建房稳态与入房提速

### [TC-ISC-10] 邀请码仅在 Host Ready 后展示
- **描述**：验证监听成功前、入口自检完成前，UI 不应展示邀请码。
- **测试步骤**：
  1. Runtime 进入 `Booting`。
  2. Runtime 进入 `ListeningReady` 但未完成入口自检。
  3. Runtime 进入 `InviteReady`。
- **预期结果**：
  - 步骤 1、2：UI 不展示可复制邀请码。
  - 步骤 3：UI 展示邀请码。

### [TC-ISC-11] 历史成功端口优先复用
- **描述**：验证 Host Runtime 优先复用上次成功的监听端口，减少端口映射波动。
- **测试步骤**：
  1. 记录上次成功端口 `7788`。
  2. 下一次建房时尝试启动监听。
- **预期结果**：
  - 若端口未冲突，优先继续使用 `7788`。

### [TC-ISC-12] 入房快速重试梯度
- **描述**：验证加入端在入口不可达时按小次数快速重试，而不是长时间无响应等待。
- **测试步骤**：
  1. 准备一个临时不可达入口的邀请码。
  2. 发起连接。
- **预期结果**：
  - 采用预设短超时与小次数快速重试。
  - 在可控时间内返回明确失败原因。
